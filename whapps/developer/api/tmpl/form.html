  • 500 – Internal Server Error

Response Headers ?

Every response from the server includes headers that you can use for advanced features. These include:

  • X-Request-ID: A unique ID for your request that can be used for debugging purposes on the API server
  • Location: When a resource is created, this is the URL that directs to the newly created resource
  • Access-Control-*: These headers allow you to bypass the same origin policy via CORS
